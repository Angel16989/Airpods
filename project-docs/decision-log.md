| 2026-07-12 | Tech stack for Apple Icon (Android app that detects connected Apple AirPods via BLE and shows an iOS-style popup with an animated AirPod visual + per-earbud/case battery percentage): Frontend/UI — Kotlin + Jetpack Compose with Material 3 theming; Backend — none, fully on-device for v0.1 (no server component); Database — Jetpack DataStore (Preferences) for small key-value state (last paired device, last known battery %, settings); Testing — JUnit + Compose UI Testing + Espresso; Deployment — Gradle build to Google Play Store (internal testing track first, then production). Options considered for the UI layer: Flutter and React Native (both rejected — BLE scanning, foreground services, and system-overlay APIs are first-class and best documented in native Android, and Compose gives declarative animation for the AirPod visual without a cross-platform bridge). Room was considered for the database and rejected as unnecessary overhead for simple key-value state. | User approved 2026-07-12 (TASK-003). | Backend Agent | Active |
| 2026-07-12 | Use local Android permissions and in-app settings as the v0.1 auth model; do not add accounts, sessions, API keys, bearer tokens, cloud auth, or a remote backend. | The approved product and feature specs are fully on-device. Bluetooth, overlay, and notification permissions give the correct platform-level gate without collecting user credentials or syncing AirPods data. | Backend Agent | Active |
