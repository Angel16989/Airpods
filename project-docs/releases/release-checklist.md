# Release Checklist

Use this before shipping a feature.

## Product

- [ ] Feature matches the agreed requirements.
- [ ] Main user flow works from start to finish.
- [ ] Empty, loading, error, and success states are handled.
- [ ] Copy is clear and free of placeholder text.

## Engineering

- [ ] Code is reviewed.
- [ ] Tests pass locally.
- [ ] New tests cover important behavior.
- [ ] Database migrations are safe and reversible where possible.
- [ ] API changes are documented.
- [ ] Feature flags are configured if needed.

## Design

- [ ] Desktop layout is checked.
- [ ] Mobile layout is checked.
- [ ] Keyboard navigation works.
- [ ] Color contrast is acceptable.
- [ ] Text does not overflow containers.

## Operations

- [ ] Analytics events are checked.
- [ ] Logs are useful for debugging.
- [ ] Rollback plan is known.
- [ ] Release notes are prepared.

## Final Sign-Off

- [ ] Product approved.
- [ ] Engineering approved.
- [ ] Design approved.
- [ ] Ready to ship.
